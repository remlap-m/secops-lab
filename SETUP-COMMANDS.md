# OIDC setup commands (reference only)

Run once, in Azure Cloud Shell, to wire GitHub Actions to Azure without any
stored secret. See README.md Step 0 for full context. No sensitive values are
recorded in this file - the actual client ID / tenant ID / subscription ID
this produces go into GitHub repo secrets (Settings -> Secrets and variables
-> Actions), never committed here.

```bash
appId=$(az ad app create --display-name "secops-lab-deployer" --query appId -o tsv)
az ad sp create --id "$appId"

az ad app federated-credential create --id "$appId" --parameters '{
  "name": "github-main",
  "issuer": "https://token.actions.githubusercontent.com",
  "subject": "repo:remlap-m/secops-lab:ref:refs/heads/main",
  "audiences": ["api://AzureADTokenExchange"]
}'

subId=$(az account show --query id -o tsv)
az group create --name RG-INFRA-LAB-EPHEMERAL --location westeurope
az role assignment create --assignee "$appId" --role Contributor \
  --scope "/subscriptions/$subId/resourceGroups/RG-INFRA-LAB-EPHEMERAL"

echo "AZURE_CLIENT_ID       = $appId"
echo "AZURE_TENANT_ID       = $(az account show --query tenantId -o tsv)"
echo "AZURE_SUBSCRIPTION_ID = $subId"
```

Copy the three output values into GitHub repo secrets as `AZURE_CLIENT_ID`,
`AZURE_TENANT_ID`, `AZURE_SUBSCRIPTION_ID`. Then add `LAB_ADMIN_PASSWORD` and
`LAB_SAFEMODE_PASSWORD` as your own strong, lab-only passwords - those are
never generated by a command, you choose them directly in the GitHub UI.

## Fix: federated credential subject mismatch (AADSTS700213)

GitHub's OIDC tokens include numeric account/repo IDs alongside the names
(`repo:owner@ACCOUNTID/repo@REPOID:ref:...`), which the original federated
credential above doesn't account for. If deploy fails with `AADSTS700213: No
matching federated identity record found`, update the credential to match the
exact subject GitHub is presenting - copy the full subject string straight out
of the workflow's error message and swap it in below.

```bash
appId=$(az ad app list --display-name "secops-lab-deployer" --query "[0].appId" -o tsv)

az ad app federated-credential update \
  --id "$appId" \
  --federated-credential-id "github-main" \
  --parameters '{"subject": "repo:remlap-m@269593897/secops-lab@1306665492:ref:refs/heads/main"}'
```

This updates the existing credential in place (same app, same trust
relationship) - it only corrects the `subject` field, nothing else changes.
Verify it landed with:

```bash
az ad app federated-credential list --id "$appId" -o table
```
